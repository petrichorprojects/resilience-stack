# Case Studies — Resilience Stack

Anonymized real engagements showing skill in production. Format below.

## Why Case Studies

Workshops without case studies are theory. Buyers (and contributing maintainers) need evidence the framework moves outcomes.

## Format

```
case-studies/
├── <skill-name>/
│   ├── case-<NN>-<short-slug>.md
│   └── ...
└── README.md
```

Each case study includes:

1. **Context** (1 paragraph) — industry / stage / trigger
2. **Symptom** — measurable pre-state
3. **Skill run** — which skill, how long, who was in the room
4. **Top 3 insights surfaced** (from deliverables)
5. **30/60/90-day outcome** — measurable post-state
6. **What we'd do differently** — facilitator notes

## Anonymization Rules

- Company → "Mid-market SaaS, vertical X" / "Series B fintech" / etc.
- Revenue → rounded to nearest $5M.
- People → roles only ("VP Sales", "Co-founder/CEO").
- Competitors → "Incumbent A", "New Entrant B".
- Customers → "Customer A/B/C" with one-line industry context.

## Contribution

Petrichor publishes 1 case study per skill per quarter. External contributors may submit anonymized case studies via PR with the above structure. Review SLA: 14 days.
